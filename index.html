AWS Account ID
Your AWS Account ID is a unique 12-digit number that identifies your AWS account. It is required to scope resources and state files correctly.
Format
Exactly 12 digits, no hyphens or spaces. Example: 123456789012
① Find it in the AWS Console
Sign in to the AWS Console, then click your account name in the top-right corner. Your 12-digit Account ID is displayed in the dropdown menu.
② Find it via the AWS CLI
Run the following command with your credentials configured:
aws sts get-caller-identity --query Account --output text

Password Policy Options
Choose how strict the IAM account password policy should be. This is applied via the aws_iam_account_password_policy resource.
AWS Default
Mirrors the AWS console defaults: min 8 chars, no complexity requirements enforced beyond that. Suitable for low-risk or internal-only accounts.
CIS Benchmark (strict)
Enforces CIS AWS Foundations controls: min 14 chars, uppercase + lowercase + numbers + symbols required, no reuse of last 24 passwords, 90-day expiry, max 5 login attempts before lockout.
None
Skips the password policy resource entirely. Use this when the account relies on SSO / Identity Center and console password login is disabled.

How to create the Terraform deploy role
Terraform needs an IAM role to assume when making AWS API calls. Follow these steps to create one and find its ARN.
① Open IAM in the AWS Console
Go to AWS Console → IAM → Roles → Create role. Choose AWS account as the trusted entity type and enter the account ID that will be running Terraform (e.g. your CI/CD account or your local credentials' account).
② Attach permissions
Attach the policies your Terraform code needs. For full bootstrapping, AdministratorAccess is common during initial setup. For production, scope it down to only the services Terraform will manage.
③ Name the role & create it
Give it a recognisable name like TerraformDeployRole. Complete the wizard and click Create role.
④ Copy the Role ARN
Click into the newly created role. At the top of the Summary panel you'll see the ARN — it looks like:
arn:aws:iam::123456789012:role/TerraformDeployRole
Paste that value into the field above.
