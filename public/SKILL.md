## Output Format

Output EXACTLY the files listed in the user specification, each preceded by its filename as `### filename` followed by a fenced code block with the `hcl` language tag.

Each file begins with exactly: `# Generated by Terraform AWS Account Setup Wizard`

Each file MUST be preceded by exactly `### filename.ext` on its own line, immediately followed by the opening code fence — no blank lines in between.

## File Placement Rules — CRITICAL

- `variables.tf` and `providers.tf` MUST each be output as their own separate `### block` with their own code fence. Do NOT merge them into main.tf or any other file.
- ALL provider blocks (including the aws provider and any default_tags configuration) MUST be placed exclusively in `providers.tf`. Do NOT include any provider block in `main.tf` or any other file — this will cause a duplicate provider configuration error in Terraform.
- ALL security resources MUST be placed exclusively in `security.tf`. Do NOT include any security resources in `main.tf` or any other file. This includes GuardDuty, CloudTrail, Security Hub, AWS Config, IAM Access Analyzer, `aws_iam_account_password_policy`, `aws_budgets_budget`, AND any supporting or ancillary resources they require — including but not limited to CloudWatch log groups, CloudWatch alarms, CloudWatch event rules, SNS topics, SNS topic policies, S3 buckets for logs, KMS keys for logging, and IAM roles or policies created specifically for these security services. If a resource exists to support a security service, it belongs in `security.tf` regardless of its resource type.
- `aws_iam_account_password_policy` MUST be in `security.tf`. NEVER place it in `main.tf` or any other file.
- Default VPC deletion resources (`aws_default_vpc`, `aws_default_subnet`, `aws_default_security_group`) MUST be in `security.tf`. NEVER place them in `main.tf`.
- `main.tf` MUST NOT contain a `terraform {}` block or any backend configuration. It contains ONLY state backend resources (S3 bucket, versioning, encryption, public access block) and data sources. The `terraform {}` block belongs exclusively in `backend.tf`.
- `backend.tf` contains a fully hardcoded backend block with all values populated from the user specification. Do NOT include commented-out alternatives, placeholders, or examples for any other backend type.
  - S3 only → full `backend "s3"` block with `bucket`, `key`, `region`, `encrypt = true`; no `dynamodb_table`
  - S3 + DynamoDB → full `backend "s3"` block with `bucket`, `key`, `region`, `encrypt = true`, and `dynamodb_table`
  - Terraform Cloud → only a `cloud {}` block
  - No commented-out code for unselected options.
